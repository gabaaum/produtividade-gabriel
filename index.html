Fontes e Metodologia — Como os dados foram obtidos

01
Datas de commit — GitHub API
Todas as datas da tabela foram extraídas via GitHub API (endpoint /repos/gabaaum/{repo}/commits?order=asc&per_page=1), buscando o primeiro commit de cada repositório. Não são datas estimadas — são os registros reais do sistema de controle de versão.
↗ github.com/gabaaum — perfil com todos os repositórios
02
Páginas Captare — repositório captare-pages
As páginas marcadas como "Captare Pages" estão hospedadas no repositório gabaaum/captare-pages no GitHub. Cada subpasta corresponde a uma entrega independente com seu próprio index.html.
↗ github.com/gabaaum/captare-pages
03
Repositórios independentes
As páginas marcadas como "Repo independente" possuem repositório próprio no GitHub, cada uma publicada via GitHub Pages. Os links "↗ ver página" na tabela acima apontam diretamente para as páginas publicadas.
↗ github.com/gabaaum?tab=repositories — lista completa de repositórios
04
Estimativa de horas — metodologia
O tempo estimado sem IA é baseado em médias de mercado para desenvolvimento de landing pages por profissional nível pleno, considerando: pesquisa, wireframe, design, desenvolvimento, testes e ajustes. As páginas foram classificadas em três categorias:
Simples
Páginas de obrigado, formulários básicos
Com IA: 0,5–1h
Sem IA: 4–6h
Média
Landing pages padrão com seções
Com IA: 2–3h
Sem IA: 10–16h
Complexa
VSL, funis completos, design pesado
Com IA: 4–5h
Sem IA: 24–40h
05
Resultado consolidado — 22 entregas
Somando as estimativas das 22 páginas: ~337h sem IA (≈ 42 dias úteis ≈ 2,1 meses) contra ~57h com IA (≈ 7 dias úteis). Razão: ~6× mais rápido. O período calendário real foi de 10/04 a 20/05/2026 (40 dias).
